Introduction

This application creates party records for testing the harvesting operation of the ARDC Party Infrastructure.

Party records are created under test sets. Each test set has a unique string that is used in the surname, person identifier and RIF-CS keys for the party records under them. This ensures that party records in each test set can be considered globally unique, which is required by some tests.

Test sets have a name and notes. These values do not appear in RIF-CS.

Party records have names, descriptions and a person identifier. That is all that is needed for testing.

Getting started

Create a new test set and then create party records under it. Then harvest the party records from the OAI-PMH feed.

When re-running tests, create another test set so that the new test party records are independent from those previously created.

OAI-PMH feed

  • Identify
  • ListSets
  • ListMetadataFormats
  • ListIdentifiers
  • ListRecords (Dublin Core)
  • ListRecords (RIF-CS)
